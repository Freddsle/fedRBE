# Quartet multiomics k-means evaluation

Generated by `../06_multiomics_kmeans.py` from the all-modality matrices in
`evaluation_data/multiomics/after/`.

The prepared matrices are already k-means-ready: each modality was row-scaled
and divided by `sqrt(n_features)` before concatenation. The central k-means
evaluation and FeatureCloud input configs therefore avoid another feature
scaling pass.

Targets (matching the four-client federation in `evaluation_data/multiomics/`):

- `condition`: donor labels `D5/D6/F7/M8` (`D6` is the limma reference donor), evaluated with `k = 4`.
- `client`: client labels `client_01_L01`, `client_02_L02`, `client_03_L05_L04`, `client_04_L03_L14`, evaluated with `k = 4`. This is the technical structure that batch correction is meant to remove. The metadata column reusing the real-dataset schema is named `lab` for compatibility with the other real-dataset scripts.

Joint samples are matched across modalities via the `pseudo_sample` keys built in `02_prepare_RBE_inputs.ipynb` (`{client}_{donor}_{rep}_{i}`); see that notebook's README for details.

Main outputs:

- `prepared/before_matrix.tsv`
- `prepared/corrected_matrix.tsv`
- `prepared/corrected_fedrbe_matrix.tsv`
- `kmeans_res/runs/1_metadata_cntrl_kmeans_res.tsv` (Before, Cor, and FedRBE cluster labels)
- `metrics/metrics_ari.tsv`
